# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

JARVIS is a voice-first AI assistant running on Windows 11. It uses Claude as its intelligence, Kokoro (local TTS) for voice synthesis, and a FastAPI/WebSocket backend with a Three.js frontend. This is a Windows port of the original macOS-only project at https://github.com/ethanplusai/jarvis.

## Setup & Run

```bat
# First-time setup (downloads source files, installs deps, generates SSL certs)
setup.bat

# Start everything (backend + frontend + opens Chrome)
start.bat
```

Manual start:
```bat
# Backend (port 8340, HTTPS)
python server.py

# Frontend (port 5173, HTTPS with proxy to backend)
cd frontend && npm run dev
```

## Architecture

```
server.py          FastAPI backend — WebSocket voice pipeline, action routing, Claude API
├── actions.py     Windows system actions (open terminal, browser, Claude Code)
├── screen.py      Windows screen capture (mss/Pillow) + window enumeration (pywin32)
├── calendar_access.py  Outlook calendar via COM (win32com)
├── mail_access.py      Outlook email via COM (win32com), read-only
├── notes_access.py     Local SQLite notes (replaces Apple Notes)
├── work_mode.py        Claude Code CLI session manager
├── memory.py           SQLite persistent memory (facts, tasks, notes)
├── browser.py          Playwright-based web browsing
├── planner.py          Task planning and clarification before Claude Code dispatch
└── dispatch_registry.py  Tracks active/recent Claude Code build jobs

frontend/src/
├── main.ts        State machine: idle → listening → thinking → speaking
├── orb.ts         Three.js particle visualization (2000 particles, audio-reactive)
├── voice.ts       Web Speech API input + Web Audio API playback
├── ws.ts          WebSocket client with auto-reconnect
└── settings.ts    Settings panel UI (API keys, preferences, system status)
```

## Key Design Patterns

**Voice pipeline**: Browser mic → Web Speech API → WebSocket → server.py → Claude API → Kokoro TTS (local) → WebSocket → browser audio playback

**Action system**: LLM embeds `[ACTION:BUILD]`, `[ACTION:BROWSE]`, etc. tags in responses. `server.py` extracts and executes them after sending the spoken response. Actions include BUILD (spawns Claude Code in Windows Terminal), BROWSE, PROMPT_PROJECT, ADD_TASK, REMEMBER, CREATE_NOTE.

**Context injection**: Every LLM call gets injected context from calendar, mail, open windows, tasks, memories, and active dispatches via `build_memory_context()` and the context-building functions in `server.py`.

**SSL**: Required for Chrome mic access. cert.pem/key.pem generated by `generate_certs.py`. Server auto-detects them. Frontend proxies through Vite.

## Windows-Specific Files

These replace the macOS originals:
- `actions.py` — Windows Terminal (`wt`) instead of AppleScript
- `screen.py` — pywin32 + mss instead of `screencapture` + AppleScript
- `calendar_access.py` — Outlook COM instead of Apple Calendar AppleScript
- `mail_access.py` — Outlook COM instead of Apple Mail AppleScript
- `notes_access.py` — SQLite instead of Apple Notes AppleScript

`apply_windows_patches.py` patches `server.py` after download to replace the 3 remaining AppleScript calls.

## Environment Variables

```
ANTHROPIC_API_KEY   Required
KOKORO_VOICE        Optional (TTS voice ID, defaults to am_fenrir)
USER_NAME           Optional (how JARVIS addresses you, default "sir")
CALENDAR_ACCOUNTS   Optional (filter Outlook accounts)
```
